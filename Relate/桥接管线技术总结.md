# MAISI + Mask 肿瘤桥接管线 — 技术总结

> 分支: `bridge-optimization` | 最后更新: 2026-06-24

---

## 一、项目背景

### 1.1 问题

MAISI (NVIDIA) 能从零生成合成 CT + 132 类分割标签，但肿瘤 mask 来自数据库查找——形状不够精确、位置不可控。我们需要在指定器官的指定位置生成**精确形状**的肿瘤。

### 1.2 解决方案：桥接管线 (Path C)

```
MAISI 步骤1                    桥接脚本                      MAISI 步骤2
生成 CT + 132类mask ──────→ 在mask上画肿瘤 ──────→ ControlNet生成含肿瘤CT
(不含肿瘤标签)               (本脚本)                     (纹理由MAISI产出)
```

**核心思路**：MAISI 的 ControlNet 完全由 132 类标签 mask 驱动。只要在 mask 上画好肿瘤形状（用对应肿瘤标签覆盖器官标签），ControlNet 就会在该区域生成肿瘤纹理。不需要 DiffTumor。

### 1.3 三个系统的角色

| 系统 | 角色 | 是否生成图像 |
|------|------|:--:|
| MAISI | 生成全部 CT 图像（器官+肿瘤纹理） | ✅ |
| Mask 项目 | 提供精确肿瘤几何形状 (椭球+弹性变形) | ❌ 只输出二值 mask |
| DiffTumor | **不参与** — MAISI ControlNet 替代其功能 | — |

## 二、管线形成过程

### 2.1 三个方案的演进

| 阶段 | 方案 | 内容 | 结论 |
|------|------|------|------|
| 最初 | Path A | MAISI → DiffTumor 纹理注入 → 融合嵌入 | 过于复杂，两套扩散模型 |
| 中期 | Path B | Mask 生成肿瘤 → Tumor 项目注入纹理 | 仍需 DiffTumor，冗余 |
| **最终** | **Path C** | **Mask 画肿瘤 → MAISI ControlNet 生成纹理** | **最简管线，一个扩散模型解决全部** |

Path C 的关键发现：MAISI 的 `tumor_adapter.py` 内部的 `TumorTextureInjector` 与我们的 DiffTumor **功能完全同构**（相同 VQGAN 编码、相同 DDPM 去噪、相同 Gaussian blending），但 MAISI ControlNet 更强大——它不需要宿主 CT，直接从 mask 生成整张 CT。

### 2.2 关键技术决策

1. **不在 mask 上裁剪肿瘤到器官内** — 保留溢出作为真实浸润特征
2. **alpha=4 而非 alpha=15** — 匹配真实肿瘤浸润深度 (2-10mm)，alpha=15 ≈ 27mm 位移远超真实值
3. **距离变换 dist³ 加权采样** — 让肿瘤中心偏向器官深处，减少大部分溢出
4. **自适应安全约束** — 极端情况（肿瘤>1.2x器官 或 overlap<50%）自动裁剪兜底

### 3.2 bridge_single() 9步管线

```
Step 1: 加载 MAISI 数据
  load_maisi_data(ct_path, label_path)
  → ct_data, label_data, affine, spacing

Step 2: 提取器官 mask
  extract_organ_mask(label_data, organ)
  → organ_mask (bool), organ_voxels
  特殊处理: kidney合并左右肾, lung合并5个肺叶, bone合并脊椎+肋骨+骨盆

Step 3: 确定肿瘤半径和位置
  radius = sample_radius(size_category, rng)
  center = sample_tumor_center(organ_mask, spacing, radius, rng, power=3.0)
  → dist³加权采样: 深处voxel权重∝dist^3, 中心偏向器官深处

Step 4: 生成肿瘤 (带重试)
  tumor_mask = create_mask(center, radius, shape, spacing, shape_config, rng)
  不裁剪到器官内部 → 保留弹性形变产生的浸润特征
  重试: 体积<200 voxels时换中心重试, 3次失败后关闭弹性形变

Step 5: 验证肿瘤质量 + 自适应安全约束
  overlap_ratio = (tumor & organ).sum() / tumor.sum()
  安全约束触发条件(满足任一):
    ① 肿瘤体积 > 1.2× 器官体积
    ② overlap < 50%
  触发 → 裁剪到器官内 (tumor &= organ_mask)

Step 6: 合并到 132 类标签
  merged = label_data.copy()
  merged[tumor_mask > 0] = tumor_label_id

Step 7: 确保 body envelope (label 200) 存在
  缺失 → CT阈值(>-200 HU)自动识别身体区域并填充

Step 8: 验证标签合法性
  所有标签 ∈ {0..132, 200}

Step 9: 保存 NIfTI
  自动命名 + 版本化 (_v2, _v3...)
```

## 四、核心技术细节

### 4.1 距离变换加权采样 (sample_tumor_center)

原始方案用均匀随机 + margin 约束采样中心，导致中心常落在器官边缘，椭球大量溢出。

**改进**：使用 `scipy.ndimage.distance_transform_edt` 计算每个体素到器官边界的距离，然后 `dist^3` 加权采样：

```python
dist = distance_transform_edt(organ_mask, spacing)  # 每个体素到边界的mm距离
distances = dist[organ_mask]                         # 只取器官内
weights = max(distances, 1.0) ** 3.0                 # dist³加权
weights = weights / weights.sum()                    # 归一化
idx = rng.choice(len(indices), p=weights)            # 加权随机
```

**效果**：6个器官的采样深度全部提升：

| 器官 | 均匀采样 avg depth | dist³采样 avg depth | 提升 |
|------|---:|---:|---:|
| liver | 8.9mm | 17.7mm | 2.0× |
| pancreas | 4.2mm | 7.5mm | 1.8× |
| kidney | 5.1mm | 8.8mm | 1.7× |
| colon | 5.2mm | 9.4mm | 1.8× |
| lung | 7.6mm | 15.2mm | 2.0× |
| bone | 3.5mm | 8.4mm | 2.4× |

### 4.2 不裁剪策略 + 自适应安全约束

**原始代码**：`tumor_mask = (raw & organ_mask)` — 裁剪到器官内，overlap 强制 100%。

**问题**：alpha=15 时胰腺 83% 被裁掉，裁剪后仅 44 voxels。

**改进**：默认不裁剪，保留溢出作为浸润特征。但加入安全网：

```python
# 极端情况才裁剪
tumor_to_organ_ratio = tumor_voxels / organ_voxels
need_constraint = (tumor_to_organ_ratio > 1.2) or (overlap_ratio < 0.5)

if need_constraint:
    tumor_mask = (tumor_mask & organ_mask).astype(np.uint8)
    overlap_ratio = 1.0
```

### 4.3 弹性形变 alpha 的医学依据

`alpha` 控制弹性形变的位移幅度（体素单位）。位移量 ≈ alpha × spacing：

| alpha | 位移 (mm, spacing≈1.7mm) | 医学依据 |
|:--:|:--:|------|
| 4 | ≈7mm | 匹配真实肿瘤浸润深度 2-10mm |
| 15 | ≈27mm | 远超真实值，导致大量溢出 |

### 4.4 verify_bridge_output.py 的 RAS 轴修复

原始代码硬编码 `maisi_shape = (shape[1], shape[2], shape[0])`，对 256×256×128 的 RAS-aligned NIfTI 产生 H≠W 的错误判断。

修复：使用 `nib.aff2axcodes(img.affine)` 判断轴方向，RAS-aligned 时直接用 nibabel shape。

### 4.5 Body Envelope 自动生成

MAISI ControlNet **依赖 label 200** 来识别身体边界。如果输入 mask 缺少 body envelope，生成质量严重下降。

桥接脚本自动检测并补全：用 CT 阈值 (>−200 HU) 识别身体区域，将未标记的身体体素设为 200。

## 五、优化历程与验证数据

### 5.1 四种策略组合对比 (10-seed 平均 overlap)

| 组合 | 策略 | liver | pancreas | kidney | colon | lung | bone |
|------|------|-------|---------|--------|-------|------|------|
| A (原始) | α=15+裁剪+均匀 | 100% | 100% | 100% | 100% | 100% | 100% |
| B | α=4+裁剪+均匀 | 100% | 100% | 100% | 100% | 100% | 100% |
| C | α=15+不裁剪+均匀 | 48% | 7.5% | 28% | 21% | 36% | 25% |
| **D (最终)** | **α=4+不裁剪+dist³** | **92%** | **69%** | **78%** | **62%** | **92%** | **79%** |

三步修改（alpha=4、不裁剪、dist³加权）必须同时生效，缺一不可。

### 5.2 当前最优结果 (20-seed, medium)

| 器官 | avg overlap | 合理范围 | 判定 | avg voxels | 约束触发率 |
|------|---:|---|---|---:|---:|
| liver | 92.6% | 90-100% | ✅ | 2,831 | 0% |
| pancreas | 79.9% | 60-80% | ✅ | 2,039 | 50% |
| kidney | 83.5% | 70-90% | ✅ | 2,168 | 25% |
| colon | 78.5% | 50-70% | ⚠️偏高 | 2,514 | 0% |
| lung | 89.6% | 85-95% | ✅ | 2,817 | 10% |
| bone | 71.2% | 40-60% | ⚠️偏高 | 2,563 | 10% |

### 5.3 器官级参数调优的负反馈陷阱

对 bone/colon/kidney/pancreas 尝试了器官级 alpha 和 center_power 覆盖，全部验证无效：

```
增大 alpha → 增大溢出 → 安全约束触发 → 裁剪到器官内 → overlap=100% → 更差
```

| 优化方向 | 预期 | 实测 | 结论 |
|----------|------|------|------|
| bone alpha=8 | 降低 overlap | 71%→79% | ❌ 更差 |
| bone 负power=-0.5 | 边缘采样 | 约束 0→60% | ❌ 溢出更严重 |
| colon alpha=6 | 降低 overlap | 约束 0→80% | ❌ 全裁剪 |
| kidney alpha=2 | 减少约束 | overlap 82→69% | ❌ 低于合理范围 |

**根本原因**：bone/colon/pancreas 的高 overlap 是解剖结构决定的（碎片/管状/太小），参数调整无法解决结构性问题。**当前 alpha=4 + dist³ + 不裁剪 + 安全约束 已是最优折中。**

## 六、使用方法

### 6.1 快速开始

```bash
# 1. 编辑 prompts.json，填入 MAISI 步骤1产出的 CT/label 路径
# 2. 运行
python bridge_maisi_mask.py
```

### 6.2 完整 MAISI 管线

```bash
# 步骤1: MAISI 生成基础 CT + mask (不含肿瘤)
python scripts/inference.py --config configs/config_infer.json

# 步骤2: 桥接 — 在mask上画肿瘤
python bridge_maisi_mask.py

# 步骤3: MAISI 根据合并mask生成最终CT
python scripts/infer_image_from_mask.py \
  --mask output/merged_label_liver_medium_tumor.nii.gz \
  --environment configs/environment_rflow-ct.json \
  --inference configs/config_infer_16g_256x256x128.json \
  --config configs/config_network_rflow.json
```

### 6.3 验证输出

```bash
# 快速格式检查
python verify_bridge_output.py --merged output.nii.gz --quick

# 完整验证
python verify_bridge_output.py --merged output.nii.gz \
  --ct original_ct.nii.gz --label original_label.nii.gz \
  --organ liver --size medium
```

## 八、已知限制与后续方向

### 8.1 结构性限制

| 器官 | 问题 | 原因 |
|------|------|------|
| bone | overlap 偏高 (71%) | 分散碎片结构，椭球全在单块骨头内 |
| colon | overlap 偏高 (78%) | 管状弯曲，深处管壁包裹椭球 |
| pancreas | large尺寸约束频繁 | 器官太小 (106cm³)，max_depth=13.6mm |

这些是解剖结构决定的，参数调优无法解决。

### 8.2 有价值的后续方向

1. **端到端验证** — 将桥接输出送入 MAISI ControlNet，评估生成 CT 的肿瘤纹理质量
2. **生成质量评估** — 用分割 Dice、HU 统计等指标量化合成 CT 的医学合理性
3. **多肿瘤合并** — 当前每个任务独立输出，支持同一 mask 上多个肿瘤需手动合并
4. **解剖结构感知采样** — 如器官连通域分析、管壁感知采样等新策略

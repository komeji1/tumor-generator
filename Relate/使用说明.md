# bridge_maisi_mask.py — 使用说明

> MAISI + Mask 项目自动桥接 — 将精确肿瘤 mask 合并到 MAISI 132 类标签

---

## 一、这个脚本做什么

MAISI 能从零生成腹部/胸部 CT + 132 类解剖标签。但它的肿瘤 mask 是数据库里查出来的——形状不够精确。

我们的 **Mask 项目** 能生成高精度的椭球+弹性变形肿瘤 mask，但需要宿主 CT 提供空间参考。

`bridge_maisi_mask.py` 把两者对接起来：

```
MAISI 生成 CT + mask (无肿瘤)
    → 本脚本在指定器官内画精确肿瘤
    → 输出合并的 132 类 mask
    → MAISI ControlNet 按合并 mask 生成最终 CT (器官+肿瘤纹理)
```

**不需要 DiffTumor**，所有图像生成都由 MAISI ControlNet 完成。本脚本只负责在 mask 上画出肿瘤的形状。

## 二、prompts.json 配置文件（主要使用方式）

仿照 Tumor 项目的设计，桥接脚本**自动检测 `prompts.json`**——直接跑即可，无需手动指定 `--config`。

### 2.1 三步上手

**① 填路径** — 打开 `work/Relate/prompts.json`，填入 MAISI 步骤1产出的两个文件：

```json
{
  "maisi": {
    "ct_path": "output/synthetic_ct.nii.gz",
    "label_path": "output/synthetic_label.nii.gz"
  },
  "tasks": [
  ]
}
```

**② 加任务** — 往 `tasks` 数组里添加要生成的肿瘤：

```json
{
  "maisi": {
    "ct_path": "E:/MAISI/output/liver_scan_ct.nii.gz",
    "label_path": "E:/MAISI/output/liver_scan_label.nii.gz"
  },
  "tasks": [
    {"organ": "liver",    "size_category": "medium", "repeat": 2},
    {"organ": "pancreas", "size_category": "small",  "repeat": 1},
    {"organ": "kidney",   "size_category": "large",  "repeat": 2},
    {"organ": "colon",    "size_category": "medium"}
  ]
}
```

**③ 运行**：

```bash
python bridge_maisi_mask.py
```

脚本自动在当前目录及脚本目录查找 `prompts.json`，找到就执行。也可以显式指定：

```bash
python bridge_maisi_mask.py --config my_other_tasks.json
```

### 2.2 prompts.json 完整字段

| 字段 | 层级 | 必需 | 说明 |
|------|------|:--:|------|
| `maisi.ct_path` | 顶层 | ✓ | MAISI 步骤1产出的 CT 文件 |
| `maisi.label_path` | 顶层 | ✓ | MAISI 步骤1产出的 132类标签文件 |
| `tasks[].organ` | 任务 | ✓ | 器官名 (liver/pancreas/kidney/colon/lung/bone) |
| `tasks[].size_category` | 任务 | ✓ | tiny / small / medium / large |
| `tasks[].repeat` | 任务 | | 重复次数 (每次随机中心+半径) |
| `tasks[].radius_mm` | 任务 | | 精确半径 mm |
| `tasks[].center_zyx` | 任务 | | 精确中心 [z,y,x] 体素坐标 |
| `tasks[].seed` | 任务 | | 任务级种子 (覆盖全局) |
| `shape_config` | 顶层 | | 自定义形状参数 (null=默认) |
| `output_dir` | 顶层 | | 输出目录 (默认: `output/`) |
| `seed` | 顶层 | | 全局随机种子 |

### 2.3 输出文件

输出默认保存到 MAISI 项目根目录下的 `output/`。`example_output/` 包含 30 个示例 `.nii.gz` 文件（来自 bridge-optimization 分支），供参考。

```

    ├── 当前目录有 prompts.json → 自动使用
```

## 四、快速开始

### 4.1 单任务模式

在 MAISI 生成的肝脏上画一个中等大小的肿瘤：

```bash
python bridge_maisi_mask.py \
  --ct output/synthetic_ct.nii.gz \
  --label output/synthetic_label.nii.gz \
  --organ liver \
  --size medium
```

输出文件自动命名：`synthetic_label_liver_medium_tumor.nii.gz`

### 4.2 批量模式

编辑配置文件 (参考 `example_bridge_config.json`)：

```json
{
  "maisi": {
    "ct_path": "output/synthetic_ct.nii.gz",
    "label_path": "output/synthetic_label.nii.gz"
  },
  "tasks": [
    {"organ": "liver",    "size_category": "medium", "repeat": 2},
    {"organ": "pancreas", "size_category": "small",  "repeat": 1},
    {"organ": "kidney",   "size_category": "large",  "repeat": 2}
  ],
  "seed": 42
}
```

```bash
python bridge_maisi_mask.py --config batch.json
```

### 4.3 完整 MAISI 管线

```bash
# 步骤1: MAISI 生成基础 CT + mask (不含肿瘤)
python scripts/inference.py --config configs/config_infer.json
# 产出: output/synthetic_ct.nii.gz, output/synthetic_label.nii.gz

# 步骤2: 桥接 — 在mask上画肿瘤
python bridge_maisi_mask.py \
  --ct output/synthetic_ct.nii.gz \
  --label output/synthetic_label.nii.gz \
  --organ liver --size medium
# 产出: output/synthetic_label_liver_medium_tumor.nii.gz

# 步骤3: MAISI 根据合并mask生成最终CT
python scripts/infer_image_from_mask.py \
  --mask output/synthetic_label_liver_medium_tumor.nii.gz \
  --environment configs/environment_rflow-ct.json \
  --inference configs/config_infer_16g_256x256x128.json \
  --config configs/config_network_rflow.json
```

## 八、肿瘤形状定制

通过 `--shape-config` 或 JSON 中的 `shape_config` 字段，可以调整肿瘤的形态特征。

默认配置 (`DEFAULT_SHAPE_CONFIG.json`)：

```json
{
  "method": "ellipsoid",
  "axis_ratio_range": [0.8, 1.2],
  "elastic_deformation": {
    "enabled": true,
    "alpha": 4.0,
    "sigma": 3.0
  },
  "center_sampling": {
    "method": "distance_weighted",
    "power": 3.0
  },
  "salt_noise": {
    "enabled": true,
    "probability": 0.02
  },
  "gaussian_filter": {
    "enabled": true,
    "sigma_mm": 1.0
  },
  "scaling_clipping": {
    "enabled": true
  }
}
```

### 参数含义

| 参数 | 作用 | 调整建议 |
|------|------|------|
| `axis_ratio_range` | 三轴椭球比例随机范围 | [0.7, 1.3] → 更扁的肿瘤 |
| `elastic_deformation.alpha` | 边界变形幅度 (体素) | 默认4.0（≈7mm位移，匹配真实浸润2-10mm）；15.0会过多溢出 |
| `elastic_deformation.sigma` | 变形平滑度 (体素) | 减小 → 更细碎的边界褶皱 |
| `center_sampling.method` | 中心采样策略 | `distance_weighted`（距离变换加权，偏向深处）；`uniform`（均匀随机） |
| `center_sampling.power` | 加权指数 | 默认3.0（强偏向深处）；0=均匀随机；2=中等偏向 |
| `salt_noise.probability` | 内部体素随机翻转比例 | 0 → 完全实心; 0.05 → 更多孔洞 |
| `gaussian_filter.sigma_mm` | 边界柔化宽度 (mm) | 增大 → 肿瘤边界更模糊 |

### 关闭所有变换 (纯椭球)

```json
{
  "elastic_deformation": {"enabled": false},
  "salt_noise": {"enabled": false},
  "gaussian_filter": {"enabled": false},
  "scaling_clipping": {"enabled": true}
}
```

## 九、输出文件

### 命名规则

自动命名格式：
```
{label文件名}_{organ}_{size_category}_tumor.nii.gz
```

示例：
```
synthetic_label_liver_medium_tumor.nii.gz
synthetic_label_pancreas_small_tumor.nii.gz
```

如果文件已存在，自动追加版本号：
```
synthetic_label_liver_medium_tumor_v2.nii.gz
synthetic_label_liver_medium_tumor_v3.nii.gz
```

### 输出内容

- **格式**: NIfTI (.nii.gz), uint8
- **形状**: 与输入 CT/label 完全一致
- **间距**: 与输入一致 (继承 affine)
- **标签**: 132 类整数标签系统
  - `0`: 背景 (空气)
  - `1`: 肝脏, `4`: 胰腺, `5`+`14`: 肾脏, ...
  - `26`: 肝脏肿瘤, `24`: 胰腺肿瘤, `116`: 肾囊肿, ...
  - `200`: Body envelope (自动补全)

## 十、Body Envelope 处理

MAISI ControlNet **依赖 label 200 (body envelope)** 来识别身体边界。缺失时会导致生成质量严重下降。

本脚本自动处理：
1. 如果输入 mask **已有** body envelope (label 200) → 直接保留
2. 如果**缺失** → 使用 CT 阈值 (>-200 HU) 识别身体区域，自动填充

输出时总是包含 body envelope。

## 十一、常见问题

### Q: 报错 "Mask 项目未找到"

确保 `work/Mask/` 目录存在。脚本在 `work/Relate/` 下运行时，自动向上查找 `../Mask`。

### Q: 生成的肿瘤太小/太大

调整 `--size` 参数，或使用 `--radius-mm` 精确指定半径。

### Q: 肿瘤有一部分在器官外面

这是正常的 — 肿瘤部分溢出器官边界代表真实的浸润生长特征。默认参数 (alpha=4 + dist³加权采样) 使 overlap 通常在医学合理范围（60-90%）。

### 自适应安全约束

当肿瘤过度溢出时自动裁剪到器官内。触发条件（满足任一）：
- 肿瘤体积 > 1.2× 器官体积（小器官+大肿瘤）
- 重叠率 < 50%（中心过于靠近边缘）

触发时终端输出 `⚠ 安全约束触发` 并显示裁剪量。绝大多数情况不触发——距离加权采样天然避免极端溢出。

### Q: 食管/子宫没有对应的肿瘤标签怎么办

MAISI 的 132 类标签系统不给食管和子宫分配独立肿瘤标签。脚本会使用器官标签本身来标识肿瘤区域，MAISI ControlNet 会尽力生成纹理（zero-shot 模式）。

### Q: 如何用同一个 mask 生成多个不同肿瘤

使用 batch 模式，在 tasks 中设置 `"repeat": N`。每次重复会随机采样不同的中心位置和半径。

### Q: 如何精确复现某个结果

使用 `--seed` 参数固定随机种子。相同的 seed + 相同的 CT/label → 一致的肿瘤位置和形状。

### Q: 输出可以直接用于 infer_image_from_mask.py 吗

可以。输出是标准的 NIfTI 132 类标签文件，直接传给：
```bash
python scripts/infer_image_from_mask.py --mask 输出文件.nii.gz
```

MAISI 的 `validate_user_mask()` 会检查 body envelope、形状、间距是否合法，不合法时自动 resample。

# MAISI + Mask 桥接 — 调试验证指南

> 本文档按**重要程度从高到低**列出验证步骤。
> 当前状态：MAISI 权重未下载，Level 1 可直接执行，Level 2-5 需完整 MAISI 环境。

## 最近修复 (2026-06-24)

**三步协同优化：alpha=4 + 不裁剪 + dist³加权采样。** 肿瘤-器官重叠率从强制100%改为医学合理的60-92%。

| 修复 | 说明 |
|------|------|
| alpha=4 | 弹性形变幅度从15降到4（≈7mm位移），匹配真实浸润深度2-10mm |
| 不裁剪 | 保留弹性形变产生的真实浸润特征（溢出器官边界部分不裁剪） |
| dist³加权采样 | `sample_tumor_center()` 使用距离变换加权让中心偏向器官深处 |
| 自适应安全约束 | 极端情况（肿瘤>1.2x器官 或 overlap<50%）自动裁剪兜底 |
| 重试+降级 | 体积<200 voxels时换中心重试，3次失败后关闭弹性形变 |

终端输出指标：
- `overlap` — 肿瘤在器官内的比例（60-92%为医学合理范围，非强制100%）
- `vox` — 肿瘤体素数
- 安全约束触发时会显示 `⚠ 安全约束触发` 并说明原因

## Level 1: 格式与标签验证（无需 MAISI）

**目的**：确认桥接输出的 NIfTI 文件格式正确，能被 MAISI 读取。

**前提**：仅需 `pip install -r requirements.txt`

### 1.1 快速格式检查

```bash
python verify_bridge_output.py --merged 输出文件.nii.gz --quick
```

期望输出全部 `[PASS]`。如果出现 `[WARN]`，检查对应项目是否可接受。

### 1.2 完整对比检查

```bash
python verify_bridge_output.py \
  --merged 输出文件.nii.gz \
  --ct 原始MAISI_CT.nii.gz \
  --label 原始MAISI_label.nii.gz \
  --organ liver --size medium
```

关键检查项：
| 检查项 | 含义 | 不通过时 |
|--------|------|------|
| 标签合法性 | 所有标签 ∈ 0..132 ∪ {200} | 桥接写了非法标签值 |
| Body Envelope | label 200 存在且比例合理 | ControlNet 质量严重下降 |
| 形状一致性 | merged shape == 原始 CT shape | 坐标错位 |
| 肿瘤-器官重叠 | 肿瘤 voxel 在器官内占比 (医学合理60-92%) | overlap<40%需注意 |
| MAISI 兼容性 | shape/spacing 是否需 resample | MAISI 会强制 resample |

### 1.3 批量 Directory 扫描

```bash
python verify_bridge_output.py -d 输出目录/ --ct ct.nii.gz --label label.nii.gz
```

## Level 2: MAISI validate_user_mask 兼容性（需 MAISI Python 环境、不需权重）

**目的**：确认 MAISI 自己的验证器能接受我们的 mask。

**前提**：
- MAISI Python 环境（MONAI, PyTorch 等）
- `infer_image_from_mask.py` 可导入
- **不需要模型权重文件** — validate 阶段只加载 NIfTI

### 2.1 运行 MAISI 内置验证

```bash
cd MAISI/
python -c "
from scripts.infer_image_from_mask import validate_user_mask
result = validate_user_mask('path/to/merged_label_liver_medium_tumor.nii.gz')
print('Shape:', result['shape'])
print('Spacing:', result['spacing'])
print('Source:', result['source_path'])
"
```

### 2.2 期望结果

```
[validate] loaded merged_label_xxx.nii.gz: shape=(256,256,128), spacing=(1.5,1.5,2.0)
[validate] ✓ all N unique label values are in the MAISI 132-class vocabulary.
[validate] ✓ shape + spacing already match a valid target — no resample needed.
```

**如果出现**：
- `mask does NOT contain label 200` → bridge 的 `ensure_body_envelope()` 未生效
- `auto-resampling` → shape/spacing 不匹配，检查 MAISI 生成参数
- `unknown label values` → 桥接写了错误标签，检查 TUMOR_LABEL_MAP

### 2.3 如果验证失败，检查原始输入

```bash
# 先验证 MAISI 自己的 mask 是否正常
python -c "
from scripts.infer_image_from_mask import validate_user_mask
print('Original MAISI mask:')
result = validate_user_mask('path/to/synthetic_label.nii.gz')
"
```

## Level 3: 单器官端到端测试（需完整 MAISI 权重）

**目的**：完整跑通 Path C — MAISI 生成 → 桥接 → MAISI 再生 → 得到含肿瘤的 CT。

**前提**：
- MAISI 全部权重 (~40GB) 已下载
- GPU 显存 ≥ 16GB
- 已按 `使用说明.md` 完成环境配置

### 3.1 选择测试器官

**推荐从肝脏开始**：
- 肝脏体积大 → 肿瘤容易放置，不会因边界问题失败
- `hepatic tumor (label 26)` 是 MAISI 训练数据中最常见的肿瘤标签
- 肝脏肿瘤在 CT 上对比度明显

### 3.2 步骤 1: MAISI 生成基础 CT + mask

```bash
cd MAISI/

# 生成一张不含肿瘤的腹部 CT
python scripts/inference.py \
  --environment configs/environment_rflow-ct.json \
  --inference configs/config_infer_16g_256x256x128.json \
  --config configs/config_network_rflow.json \
  --body_region abdomen \
  --anatomy_list liver \
  --controllable_anatomy_size "[]" \
  --output_size 256 256 128 \
  --spacing 1.5 1.5 2.0 \
  --num_output_samples 1

# 记录输出路径:
#   CT:    output/synthetic_ct.nii.gz  (或类似命名)
#   Label: output/synthetic_label_full.nii.gz
```

### 3.3 步骤 2: 桥接 — 画肿瘤

编辑 `Relate/prompts.json`：

```json
{
  "maisi": {
    "ct_path": "../MAISI/output/synthetic_ct.nii.gz",
    "label_path": "../MAISI/output/synthetic_label_full.nii.gz"
  },
  "tasks": [
    {"organ": "liver", "size_category": "medium", "repeat": 1}
  ],
  "seed": 42
}
```

```bash
cd Relate/
python bridge_maisi_mask.py
# 输出: synthetic_label_full_liver_medium_tumor.nii.gz
```

**检查点**：终端输出中确认以下信息合理
- `organ_voxels` > 1000 (器官不能太小)
- `tumor_voxels` 在合理范围 (medium ≈ 4,000–34,000 voxels)
- `has_body_envelope: True`

### 3.4 步骤 3: MAISI 根据合并 mask 重新生成 CT

```bash
cd MAISI/

python scripts/infer_image_from_mask.py \
  --mask ../Relate/synthetic_label_full_liver_medium_tumor.nii.gz \
  --environment configs/environment_rflow-ct.json \
  --inference configs/config_infer_16g_256x256x128.json \
  --config configs/config_network_rflow.json \
  --output_dir output/bridged/
```

### 3.5 步骤 4: 对比验证

```bash
# 用 Python 快速对比两张 CT 的差异
python -c "
import nibabel as nib
import numpy as np

# 加载
ct_before = nib.load('output/synthetic_ct.nii.gz').get_fdata()
ct_after  = nib.load('output/bridged/synthetic_ct.nii.gz').get_fdata()
label_before = nib.load('output/synthetic_label_full.nii.gz').get_fdata()
label_after  = nib.load('../Relate/synthetic_label_full_liver_medium_tumor.nii.gz').get_fdata()

# 检查 mask 差异
tumor_label = 26
tumor_before = (label_before == tumor_label).sum()
tumor_after  = (label_after == tumor_label).sum()
print(f'肿瘤 label 26 voxels: 桥接前={tumor_before}, 桥接后={tumor_after}')
assert tumor_after > 0, 'FAIL: 桥接后没有肿瘤标签！'
assert tumor_before == 0, 'WARN: MAISI 原始 mask 已有肿瘤标签'

# 检查 CT 在肿瘤区域是否有变化
tumor_mask = (label_after == tumor_label)
diff = np.abs(ct_after[tumor_mask] - ct_before[tumor_mask])
print(f'肿瘤区域 CT 差异: mean={diff.mean():.1f} HU, max={diff.max():.1f} HU')
if diff.mean() < 10:
    print('WARN: 肿瘤区域 CT 几乎无变化，ControlNet 可能未识别肿瘤标签')
else:
    print('OK: 肿瘤区域 CT 有明显变化')

# 检查形状一致性
assert ct_before.shape == ct_after.shape, 'FAIL: CT 形状不一致'
print(f'形状一致: {ct_before.shape}')

print('\\n--- 基础检查完成 ---')
"
```

**期望**：
- `tumor_after > 0`（桥接成功写入肿瘤标签）
- `tumor_before == 0`（原始 MAISI mask 不含肿瘤，符合预期）
- `ct_before.shape == ct_after.shape`（形状不变）
- 肿瘤区域 CT 有显著 HU 差异（说明 ControlNet 响应了肿瘤标签）

### 3.6 目视检查 (推荐用 ITK-SNAP 或 3D Slicer)

```bash
# ITK-SNAP: 同时打开两张 CT 对比
itksnap -g output/synthetic_ct.nii.gz \
        -o output/bridged/synthetic_ct.nii.gz \
        -s output/synthetic_label_full.nii.gz
```

**观察要点**：
- 肿瘤区域是否出现与周围器官不同的纹理
- 肿瘤边界是否平滑（不应该是锐利边界 — MAISI ControlNet 会做纹理过渡）
- Body envelope 区域是否正常（不应该出现异常空洞）

## Level 5: 边界情况测试（需完整 MAISI 权重）

### 5.1 小器官 (肿瘤可能超出边界)

```json
{"organ": "pancreas", "size_category": "large", "repeat": 1}
```

检查点：桥接会约束肿瘤到器官内（重叠率 100%），裁剪掉的溢出部分会在终端显示百分比。如果裁剪率 >50%，建议关闭弹性形变。

### 5.2 同一 CT 上多个肿瘤

```json
{"organ": "liver", "size_category": "medium", "repeat": 3}
```

检查点：所有肿瘤标签都存在于输出 mask 中。如果 repeat 在同一输出 mask 上（目前每个任务独立输出），确认不会互相覆盖。**当前每个任务是独立输出文件，如需同一 mask 多肿瘤需手动合并。**

### 5.3 Zero-shot 器官

```json
{"organ": "esophagus", "size_category": "tiny", "repeat": 1},
{"organ": "uterus",    "size_category": "small", "repeat": 1}
```

检查点：桥接应给出 ⚠ "无独立肿瘤标签" 警告但仍产出 mask。MAISI ControlNet 使用器官标签本身作为条件——肿瘤纹理质量不可预期。

### 5.4 MAISI mask 已含肿瘤时

如果 MAISI 输出了一个本身已有肿瘤标签的 mask，再画肿瘤上去：

```bash
# 检查原始 mask 中是否已有肿瘤标签
python -c "
import nibabel as nib
label = nib.load('synthetic_label_full.nii.gz').get_fdata()
tumor_labels = [23,24,26,27,116,128]
for tl in tumor_labels:
    count = (label == tl).sum()
    if count > 0:
        print(f'Label {tl}: {count} voxels — MAISI 已有此肿瘤')
"
```

检查点：桥接应覆盖原有肿瘤标签（`merged[tumor_mask > 0] = tumor_label_id`）。

### 5.5 不同 MAISI 输出尺寸

MAISI 支持多种尺寸：`256x256x128`, `384x384x256`, `512x512x384` 等。

```bash
# 测试大尺寸
python scripts/inference.py \
  --output_size 384 384 128 --spacing 1.0 1.0 2.0 ...
python bridge_maisi_mask.py --config prompts_large.json
python verify_bridge_output.py --merged ...
```

检查点：不同尺寸下 verify 是否都通过 MAISI 兼容性检查。

## 快速诊断脚本

将以下脚本保存为 `diagnose.sh` (Linux/Mac) 或 `diagnose.bat` (Windows)，一键运行 Level 1-2 检查：

```bash
#!/bin/bash
# diagnose.sh — 桥接输出一键诊断
# 用法: bash diagnose.sh <merged_mask.nii.gz> [ct.nii.gz] [label.nii.gz]

MERGED="$1"
CT="${2:-}"
LABEL="${3:-}"

echo "============================================"
echo " MAISI + Mask Bridge — 诊断报告"
echo "============================================"
echo "Mask: $MERGED"
echo "CT:   ${CT:-'(未提供)'}"
echo "Label: ${LABEL:-'(未提供)'}"
echo ""

# Level 1: 格式检查
echo "--- Level 1: 格式与标签 ---"
python verify_bridge_output.py --merged "$MERGED" --quick

# Level 1+: 如果有 CT/label, 做完整检查
if [ -n "$CT" ] && [ -n "$LABEL" ]; then
    echo ""
    echo "--- Level 1+: 空间一致性 ---"
    python verify_bridge_output.py --merged "$MERGED" --ct "$CT" --label "$LABEL"
fi

# Level 2: MAISI validate (如果 MAISI 环境可用)
if python -c "from scripts.infer_image_from_mask import validate_user_mask" 2>/dev/null; then
    echo ""
    echo "--- Level 2: MAISI validate_user_mask ---"
    python -c "
from scripts.infer_image_from_mask import validate_user_mask
import sys
try:
    result = validate_user_mask('$MERGED')
    print('OK: MAISI 验证通过')
    print('  shape:', result['shape'])
    print('  spacing:', result['spacing'])
except Exception as e:
    print('FAIL:', e)
    sys.exit(1)
"
else
    echo ""
    echo "--- Level 2: 跳过 (MAISI 环境不可用) ---"
fi

echo ""
echo "============================================"
echo " 诊断完成"
echo "============================================"
```

## 问题排查速查表

| 现象 | 可能原因 | 检查 |
|------|------|------|
| `ModuleNotFoundError: nibabel` | 未装依赖 | `pip install -r requirements.txt` |
| `Mask 项目未找到` | Mask 不在预期位置 | 设置 `MASK_PROJECT_DIR` 或 `git worktree add` |
| `器官 xxx 体素过少` | MAISI 生成的器官太小 | 换一张 CT 或换一个更大的器官 |
| `肿瘤仅 X% 在器官内` | 旧版 bug，修复后不再出现 | 升级到最新 bridge_maisi_mask.py |
| MAISI `does NOT contain label 200` | Body envelope 缺失 | bridge 的 `ensure_body_envelope()` 未执行，检查 CT threshold |
| MAISI `unknown label values` | 标签超出 0..132 | 检查 `TUMOR_LABEL_MAP` 是否正确 |
| MAISI 自动 resample | shape/spacing 不匹配 | MAISI 会处理但质量下降；用与 `_is_valid_target` 兼容的参数 |
| 生成的 CT 肿瘤区域无变化 | ControlNet 未响应标签 | 确认 label 200 存在；尝试增大肿瘤体积；确认模型已正确加载 |
| 肿瘤纹理不真实 | ControlNet 对该标签训练不足 | 特定肿瘤标签的生成质量因训练数据而异；尝试增大肿瘤尺寸 |

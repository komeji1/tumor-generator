# MAISI + Mask 项目对接方案 — 途径 C 实现

> **目标**: MAISI 生成器官底图 → Mask 项目画精确肿瘤 → MAISI 根据合并mask生成完整 CT
> **不参与**: DiffTumor (Tumor 项目)

---

## 一、整体流程

```
MAISI 步骤1: 生成 CT + 132类 mask (不含肿瘤, controllable_anatomy_size=[])
     │
     ▼
提取器官mask: 从 132 类 mask 中取 label==1 (肝脏), label==4 (胰腺) 等
     │
     ▼
Mask 项目: 在器官mask内即时生成肿瘤
  调用 create_mask(center_zyx, radius_mm, organ_shape, spacing, shape_config)
  返回: 二值肿瘤mask (H,W,D), 值域 {0,1}
     │
     ▼
合并mask: 肿瘤区域的 voxel 标签改为对应肿瘤ID
  肝脏肿瘤: 1 → 26
  胰腺肿瘤: 4 → 24
  肾囊肿:   5/14 → 116
  结肠癌:   62 → 27
     │
     ▼
保底处理: 如果无 body envelope (label 200), 自动生成
     │
     ▼
MAISI 步骤2: infer_image_from_mask.py --mask merged_mask.nii.gz
  → ControlNet 根据完整132类mask生成最终CT
```

## 二、两个项目的交接格式

### MAISI 产出 (步骤1 的输出)

```python
# MAISI 的 sample_multiple_images() 输出:
synthetic_ct.nii.gz        # 合成CT, 形状 (H,W,D), 间距 ~1.5mm
synthetic_label.nii.gz     # 132类标签, 形状 (H,W,D), 整数 {0,1,...,200}
```

### Mask 项目需要这俩

```python
# 传给 create_mask() 的参数:
center_zyx:  (z, y, x)     # 肿瘤中心 (voxel坐标)
radius_mm:   float         # 肿瘤半径 (mm)
shape:       (D, H, W)     # 来自合成CT的形状
spacing:     (dz, dy, dx)  # 来自合成CT的affine
shape_config: dict         # 椭球/弹性变形/salt noise等参数
```

### 合并后返回给 MAISI (步骤2 的输入)

```python
merged_label.nii.gz         # 132类整数标签
# 形状: (H,W,D), 间距: 与合成CT一致
# 内容: 原始器官label + 我们的肿瘤label + body envelope (200)
```

## 三、核心对接函数

```python
def bridge_maisi_to_mask(
    maisi_ct_path: str,           # MAISI 生成的 CT
    maisi_label_path: str,        # MAISI 生成的 132 类 mask
    organ_name: str,              # 目标器官
    size_category: str,           # tiny/small/medium/large
    shape_config: dict = None,    # Mask 项目的 mask 生成配置
    rng: np.random.Generator = None,
) -> str:
    """
    对接 MAISI 输出到 Mask 项目，生成合并的 132 类 mask。
    
    Returns: merged_label_path (可直接传给 MAISI infer_image_from_mask.py)
    """
    
    # 1. 加载 MAISI 数据
    ct_img = nib.load(maisi_ct_path)
    label_img = nib.load(maisi_label_path)
    ct_data = ct_img.get_fdata()       # (H,W,D)
    label_data = label_img.get_fdata().astype(np.int32)  # (H,W,D)
    affine = ct_img.affine
    spacing = _get_spacing(affine)     # (dz, dy, dx)
    shape = ct_data.shape              # (H,W,D)
    
    # 2. 提取器官 mask
    ORGAN_LABEL = {"liver": 1, "pancreas": 4, "kidney": 5, "colon": 62}
    organ_label_id = ORGAN_LABEL[organ_name]
    organ_mask = (label_data == organ_label_id)  # (H,W,D) bool
    
    if organ_mask.sum() == 0:
        raise ValueError(f"No {organ_name} (label={organ_label_id}) found in MAISI mask")
    
    # 3. 调用 Mask 项目生成肿瘤
    center_zyx = _sample_tumor_center(organ_mask, spacing, size_category)
    radius_mm = _sample_radius(size_category, rng)
    
    from mask_generator import create_mask
    tumor_mask = create_mask(
        center_zyx=center_zyx,
        radius_mm=radius_mm,
        shape=shape,            # (H,W,D) -- 和 MAISI CT 一致
        spacing=spacing,
        shape_config=shape_config or DEFAULT_SHAPE_CONFIG,
        rng=rng,
    )  # 返回 (H,W,D) uint8, {0,1}
    
    # 4. 合并: 肿瘤区域改标签
    TUMOR_LABEL = {"liver": 26, "pancreas": 24, "kidney": 116, "colon": 27}
    tumor_label_id = TUMOR_LABEL[organ_name]
    
    merged = label_data.copy()
    merged[tumor_mask > 0] = tumor_label_id
    
    # 5. 保底: 加 body envelope
    if 200 not in merged:
        merged[merged == 0] = 200  # body envelope 覆盖背景
        merged[tumor_mask > 0] = tumor_label_id  # 恢复肿瘤
        merged[label_data > 0] = label_data[label_data > 0]  # 恢复器官
    
    # 6. 保存
    output_path = maisi_label_path.replace(".nii.gz", f"_{organ_name}_tumor.nii.gz")
    nib.save(nib.Nifti1Image(merged.astype(np.uint8), affine), output_path)
    
    return output_path
```

## 四、三个文件改动清单

### 4.1 新建文件

| 文件 | 位置 | 说明 |

| `bridge_maisi_mask.py` | `work/Relate/` | 核心桥接模块，含上述函数 |

| 文件 | 改动 |
|------|------|
| `infer_image_from_mask.py` | 不改。直接用 `--mask merged_label.nii.gz` |

MAISI 完全不动——它本来就支持 `--mask` 参数接受外部 mask 文件。

### 4.3 Mask 项目侧改动

| 文件 | 改动 |
|------|------|
| `Step5/src/mask_generator.py` | 不改。`create_mask()` 接口已经满足需求 |

Mask 项目也不动——`create_mask()` 接受任意 shape/spacing/organ_mask，不需要知道宿主 CT 是真实还是合成的。

## 五、使用流程

```bash
# 1. MAISI 生成不含肿瘤的 CT + mask
python scripts/inference.py --config configs/config_infer.json

# 2. 桥接: 在 MAISI mask 上画肿瘤
python bridge_maisi_mask.py \
  --ct output/synthetic_ct.nii.gz \
  --label output/synthetic_label.nii.gz \
  --organ liver \
  --size medium

# 3. MAISI 根据合并 mask 重新生成带肿瘤的 CT
python scripts/infer_image_from_mask.py \
  --mask output/synthetic_label_liver_tumor.nii.gz \
  --config configs/config_infer.json
```

## 六、需要确认的前置条件

1. **MAISI 能生成不含肿瘤的器官 mask**。`controllable_anatomy_size=[]` 模式使用数据库查找，肿瘤标签由是否存在肿瘤 mask 决定。如果数据库里所有肝脏 mask 都有肿瘤，需要确认能否过滤掉。
2. **MAISI 的 body envelope (label 200)** 是否在所有输出中都存在——如果没有，桥接脚本补上。
3. **Mask 项目的 shape_config** 参数是否需要在 MAISI 的体素间距下调整（alpha/sigma 是体素单位，不同 spacing 效果不同）。
4. **infer_image_from_mask.py 的 --mask 参数** 验证逻辑是否能接受我们合并后的 mask（形状/间距匹配原 MAISI 输出即可通过 `validate_user_mask()`）。

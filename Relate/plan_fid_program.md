# FID 真实性评估程序计划

## 目标
写一个完整的程序，严格对标 MAISI 论文的 FID 评估方法，计算合成含肿瘤CT的真实性。

## 正确的对比逻辑
- **真实参考集**: autoPET50（含肿瘤的真实CT）
- **合成待评集**: 48例含肿瘤的合成CT
- **FID含义**: 合成CT与真实CT的分布距离，越低越好

## 当前问题需要解决

### 1. 协方差矩阵奇异
48例在2048维空间下协方差矩阵不满秩 → sqrtm计算不准 → FID值不可靠

解决方案:
- 用 **SqueezeNet** (1000维) 作为辅助特征网络，维度更低，48例足够覆盖
- RadImageNet ResNet50 (2048维) 也保留，但标注其统计可靠性
- 报告两种网络的FID，让用户判断

### 2. 预处理参数与论文不一致
论文: 512³, center_slices_ratio=0.4, resampling 1mm³
我们之前: 256×256×128

解决方案:
- 默认使用 256×256×128 (8GB VRAM友好)
- 提供 --paper_config 快捷参数，一键切换到论文配置 512×512×512 (需要先crop再resample避免OOM)
- 两种配置都运行，报告对比结果

### 3. 数据泄露风险
autoPET 可能与 MAISI 训练数据重叠

解决方案:
- 在程序输出中明确标注数据来源和潜在泄露风险
- 程序本身无法解决泄露问题（需要独立参考集），但做好标注

### 4. 协方差奇异的可视化诊断
- 计算特征矩阵的奇异值分布，报告有效秩
- 如果有效秩远低于特征维度，标注FID不可靠

## 程序结构

```
Relate/compute_fid_single_gpu.py  (改造现有脚本)
```

改造要点:
1. 新增 `--diagnosis` 参数: 报告特征矩阵的奇异值分布和有效秩
2. 新增 `--paper_config` 快捷参数: 一键设置512³ + center_slices=0.4 + 1mm³ resampling
3. 新增 SqueezeNet 作为 fallback 特征网络（与RadImageNet并列报告）
4. 输出中明确标注: 对比逻辑、数据来源、统计可靠性警告
5. 修正输出格式: 不再与"无肿瘤CT论文值"对比，而是明确写"含肿瘤真实CT vs 含肿瘤合成CT"
